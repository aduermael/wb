# Release

This is maintainer documentation for publishing `wb` releases.

Releases are intentionally built from a local Mac for now. There is no GitHub Action because the release binaries need to be built on macOS 26+.

## Requirements

Build requirements:

- A Mac with macOS 26.0 or newer
- Xcode 26 or newer, with Swift 6.2 SDKs

Publish requirements:

- `gh` authenticated to `aduermael/wb`
- `gh` access to `aduermael/homebrew-tap`
- the release assets generated by `./release.sh <version> --build-only`
- the same clean checkout and commit used for the build

## All In One

Use this only on a Mac that also has `gh` authenticated:

```bash
./release.sh 0.1.0
```

The script builds `arm64` and `x86_64` release binaries, packages tarballs, writes SHA-256 checksums, pushes `main`, creates/pushes an annotated tag like `v0.1.0`, and creates or updates the GitHub Release.

It also updates the Homebrew formula in `aduermael/homebrew-tap` when both macOS release assets are present. To publish without touching the tap:

```bash
./release.sh 0.1.0 --no-tap
```

To use another tap repository:

```bash
./release.sh 0.1.0 --tap-repo owner/homebrew-tap
```

## Split Build And Publish

If the Mac does not have `gh`, build on the Mac:

```bash
./release.sh 0.1.0 --build-only
```

That creates:

```text
dist/v0.1.0/wb-macos-arm64.tar.gz
dist/v0.1.0/wb-macos-arm64.tar.gz.sha256
dist/v0.1.0/wb-macos-x86_64.tar.gz
dist/v0.1.0/wb-macos-x86_64.tar.gz.sha256
```

Copy `dist/v0.1.0/` to an environment with `gh` authenticated to `aduermael/wb`, using the same checkout and commit, then publish:

```bash
./release.sh 0.1.0 --publish-only
```

To release only one architecture:

```bash
./release.sh 0.1.0 --build-only arm64
./release.sh 0.1.0 --publish-only arm64
```
